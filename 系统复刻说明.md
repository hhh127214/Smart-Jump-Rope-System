# AI跳绳（初级版 Web）——系统复刻说明（单摄像头 · 5人并发）

本文档用于完整描述当前系统的结构、流程、接口与关键实现细节，目标是让另一个 AI/开发者仅依据本文档即可实现与当前版本“看起来与行为一致”的系统（当前摄像头识别/动作检测/计数为占位模拟，流程完整可走通）。

---

## 1. 系统目标与边界

### 1.1 目标
- 单摄像头覆盖 5 个固定站位（站位1~5），支持 5 人并发测试的全流程演示
- 全流程：入口（开始识别/摄像头设置）→ 扫脸识别（占位）→ 5人信息确认（纵向排布）→ 站位绑定（依次点名1~5做指定动作，占位）→ 测试中同屏计数（占位）→ 结果页汇总 + 落库 + 导出

### 1.2 当前版本的占位点（必须说明）
- “扫脸识别”占位：后端从 SQLite 的 `users` 表取前 5 个用户填充到 5 个站位
- “指定动作检测（挥手/举手/跳一下）”占位：前端按钮触发 `POST /api/binding/gesture`
- “计数算法”占位：后端线程按不同节奏递增 5 路计数
- “离开ROI/遮挡/跟踪丢失”占位：后端在 running 阶段按时间取模生成状态

## 4. 页面与流程（前端）

前端为“单页多视图”，通过 `<section class="view" data-view="...">` 切换显示。

### 4.1 视图列表（data-view 必须一致）
- `home`：入口页（开始识别 / 摄像头设置）
- `settings`：摄像头设置页（占位配置）
- `recognition`：扫脸识别（占位）+ 5人信息列表（纵向排布）+ 逐人确认
- `binding`：身份确认与站位绑定（点名1~5）+ 开始按钮
- `running`：测试中同屏计数（5路）+ 剩余时间/排名/状态 + 结束按钮
- `result`：结果页（5人汇总、排名、导出、重新开始）

### 4.2 页面交互与按钮（按顺序）

#### 第1页：入口（home）
- 两个按钮：
  - `开始识别`（`#btnGoRecognize`）：切到 `recognition`
  - `摄像头设置`（`#btnGoSettings`）：切到 `settings`
- 右上角 `#btnHome`：强制回 `home`（不改变后端流程）

#### 第2页：扫脸识别 + 5人信息（recognition）
- 左侧：
  - `开始扫脸识别`（`#btnStartRecognition`）→ `POST /api/recognition/start`
  - `重新识别`（`#btnRedoRecognition`）→ 再次 `POST /api/recognition/start`
- 右侧纵向信息列表 `#peopleList`
  - 每个站位显示：姓名/学号/班级/性别 + `确认`按钮
  - 单人确认：`POST /api/slots/confirm_info`（slot=1..5）
- 全部 5 人确认后：
  - `确认并进入站位确认`（`#btnConfirmInfoNext`）可点击 → 切到 `binding`

#### 第3页：身份确认/站位绑定（binding）
- 后端进入 `binding` 阶段后会给出点名站位 `promptSlot`（1..5）
- 右侧站位卡片 `#bindSlots`
  - 只有当前被点名站位的 `挥手确认`按钮可点
  - 点击后：`POST /api/binding/gesture`（slot=promptSlot）
  - 完成 1→5 后进入 `ready`
- `开始`按钮（`#btnStartTest`）：
  - 仅 `ready` 时可点击
  - 点击：`POST /api/test/start`（durationSec 来自页面下拉框），随后切到 `running`

#### 第4页：测试中（running）
- 展示：
  - 同屏 5 路计数
  - 剩余时间（`remainingSec`）
  - 排名（`ranking`）
  - 异常提示（`slotStatus`：OUT_OF_ROI/OCCLUDED/TRACK_LOST）
- `结束`按钮（`#btnStop`）：
  - `POST /api/test/stop` → 切到 `result`

#### 第5页：结果（result）
- 展示站位1~5最终次数与排名
- `导出结果`（`#btnExport`）：
  - 前端生成 JSON 并下载（不依赖后端导出接口）
- `重新开始`（`#btnRestart`）：
  - `POST /api/system/reset`，回到入口页

## 5. 轮询机制与“闪退修复”（关键）

前端每 ~240ms 轮询 `GET /api/test/metrics` 以驱动 UI 状态刷新。

为避免“点击开始识别后被 idle 状态拉回首页”的闪屏问题：
- 自动导航策略为：只允许从当前页“向后续流程页前进”，不允许回退（settings 页也不会被跳走）
- 这依赖 `app.js` 中的 `viewStage(...)` 与 `refreshMetricsOnce(...)` 的比较逻辑

## 6. 后端：状态机、线程与静态资源

后端实现文件：[server.py](file:///c:/Users/17128/Desktop/%E8%B7%B3%E7%BB%B3%E7%B3%BB%E7%BB%9F/server.py)

### 6.1 静态资源服务
- `GET /` → `index.html`
- `GET /styles.css` → `styles.css`
- `GET /app.js` → `app.js`

### 6.2 流程状态机（FLOW_STATE）
核心字段（线程安全，受 `FLOW_LOCK` 保护）：
- `phase`：`idle | confirming_info | binding | ready | running | result`
- `slots`：长度 5，每项：
  - `slot`：1..5
  - `user`：用户对象（`id,name,className,gender,studentId,hasFace`）
  - `confirmedInfo`：bool
  - `confirmedPosition`：bool
- `prompt_slot`：点名站位 1..5（绑定完成后为 0）
- `run_id`：一次测试的 run id（`r_...`）
- `duration_sec`：测试时长
- `started_at_ms / ended_at_ms`
- `counts`：长度 5 的计数
- `points`：长度 5 的曲线点（每点 `{t,c}`）
- `cadences_ms`：长度 5，不同节奏模拟计数速度

### 6.3 模拟计数线程（flow_worker）
- 当 `phase == running` 时按 `cadences_ms` 递增 `counts[i]`
- 自动到时：duration 到达后进入 `result`

### 6.4 模拟摄像头画面（SVG）
- `GET /api/camera/frame.svg`
- 输出：一张 SVG，带 5 个站位框与计数
- 若在 binding 阶段，会高亮 `prompt_slot` 对应的站位框

### 6.5 模拟异常状态
- `GET /api/test/metrics` 中返回 `slotStatus`（长度 5）
- 在 running 阶段按时间变化模拟：
  - `OK`
  - `OUT_OF_ROI`
  - `OCCLUDED`
  - `TRACK_LOST`

## 7. API 契约（必须一致）

### 7.1 健康检查
- `GET /api/health` → `{ ok: true }`

### 7.2 摄像头配置（占位）
- `GET /api/camera/config` → `{ config: { mode: "SIM"|"USB", index: number } }`
- `POST /api/camera/config` body: `{ mode, index }` → `{ ok: true, config }`

### 7.3 系统重置
- `POST /api/system/reset` → `{ ok: true, state: FLOW_STATE }`

### 7.4 开始识别（占位）
- `POST /api/recognition/start` → `{ ok: true, slots: [...] }`
  - 行为：从 DB `users` 取前 5 人填充 slots，并将 phase 置为 `confirming_info`

### 7.5 确认信息
- `POST /api/slots/confirm_info` body: `{ slot: 1..5, confirmed: true }` → `{ ok: true, state }`
  - 5 人全部 confirmed 后：phase 自动变为 `binding`，`prompt_slot=1`

### 7.6 站位绑定（指定动作占位）
- `POST /api/binding/gesture` body: `{ slot: 1..5 }`
  - 如果 slot != prompt_slot：409 `{ error:"NOT_PROMPTED", promptSlot }`
  - 依次完成 1..5 后：phase 变为 `ready`，prompt_slot=0

### 7.7 开始测试
- `POST /api/test/start` body: `{ durationSec }`
  - 仅 `phase == ready` 可开始，否则 409 `{ error:"NOT_READY" }`
  - 开始后 phase=running，生成 run_id，并初始化计数

### 7.8 停止测试
- `POST /api/test/stop`
  - running→result，并将数据写入数据库

### 7.9 轮询指标（前端驱动核心）
- `GET /api/test/metrics` → JSON：
  - `phase`：流程阶段
  - `runId`
  - `durationSec`
  - `remainingSec`
  - `counts`：长度 5
  - `ranking`：数组 `[{slot,count}, ...]`
  - `promptSlot`
  - `slots`：长度 5（用于前端渲染人员信息/站位）
  - `slotStatus`：长度 5（OK/OUT_OF_ROI/OCCLUDED/TRACK_LOST）
  - `ts`
