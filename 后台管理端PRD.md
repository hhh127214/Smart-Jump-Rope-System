## 1. 项目背景与目标

### 1.1 现有系统概况

当前系统已具备以下功能模块：

- **大屏端（Screen）**：基于 Vue 3 + Vite 的单页面应用，实现 5 人并发跳绳测试的全流程，包括入场 → 人脸识别 → 站位绑定 → 倒计时 → 实时计数 → 成绩展示。
- **后端 API**：基于 Python FastAPI，提供测试流程接口（`/api/*`），已包含流程状态机、SQLite 数据库初始化、模拟计数线程等。
- **数据库（SQLite）**：已建表并包含种子数据，涵盖学校、年级、班级、学生、人脸样本、考场、设备、任务、测试记录、成绩、异常、设置、操作日志、登录日志、流程日志等。

### 1.2 后台管理端目标

在现有大屏端基础上，新增**后台管理端（Admin Panel）**，为管理员/教师提供对全系统的集中管理能力。后台管理端与大屏端共享同一后端 API，独立部署前端。

### 1.3 技术约束

- 后台前端框架：Vue 3（与现有大屏端一致，便于共用组件与构建体系）
- API 约定：复用现有 FastAPI 后端，新增 `/api/admin/*` 路由组
- 数据库：继续使用 SQLite（`rope.db`），后续可平滑迁移至 PostgreSQL
- 无需引入重量级 UI 框架，优先轻量实现

## 2. 用户角色与权限

### 2.1 角色定义

| 角色     | 权限范围                                                               |
|----------|------------------------------------------------------------------------|
| 超级管理员 | 全部功能，包括系统设置、用户管理、数据库操作                            |
| 学校管理员 | 管理本学校范围内的年级、班级、学生、考场、设备、任务、成绩、异常        |
| 教师     | 查看所教班级的学生、成绩；管理测试任务；导出本班成绩                    |

### 2.2 认证方式

- 用户名 + 密码登录
- JWT Token 认证（后端新增 `/api/admin/auth/login`）
- Token 有效期 24 小时，支持刷新

## 3. 功能模块

### 3.1 登录与首页

#### 3.1.1 登录页
- 输入：用户名、密码
- 登录成功后跳转至仪表盘
- 失败提示错误信息，记录到 `login_logs`
- 对应数据库表：`login_logs`

#### 3.1.2 仪表盘（Dashboard）
- **统计卡片**：学校总数、学生总数、今日测试人次、累计测试人次
- **近期测试趋势**：近 7/30 天测试人次折线图
- **最新测试记录**：展示最近 10 条测试记录，包含时间、考场、参与人数
- **异常告警**：最近 7 天未处理的异常数量与列表
- **设备状态概览**：在线/离线设备数量

| 编辑任务       | 修改任务信息                                        |

### 3.10 成绩管理

对应数据库表：`scores`, `test_runs`, `test_run_slots`

#### 3.10.1 成绩列表

- 分页展示所有测试成绩
- 支持筛选：学校、年级、班级、学生、考场、任务、日期范围
- 列表字段：学生姓名、学号、班级、考场、站位、最终次数、状态、审核状态、测试时间
- 排序：按测试时间倒序

#### 3.10.2 成绩详情

- 测试基本信息：任务名称、考场、时间、时长
- 5 个站位的详细计数与趋势曲线（`points_json` 数据可视化）
- 异常记录列表（关联 `exceptions` 表）
- 审核操作区

#### 3.10.3 成绩审核

- 状态标签：`normal` / `exception`
- 审核状态：`none` / `pending` / `approved` / `rejected`
- 审核操作：确认成绩有效、标记异常、添加备注（`handle_remark`）
- 批量审核：支持勾选多条记录一键审核

#### 3.10.4 成绩导出

- 按当前筛选条件导出为 Excel/CSV
- 导出字段可配置（默认全字段）
- 支持按班级导出汇总报表（班级平均成绩、最高/最低成绩）

#### 3.10.5 成绩统计

- 按班级/年级/学校维度统计：
  - 平均次数、最高次数、最低次数、参与人数
  - 及格率、优秀率（可配置阈值）
- 趋势分析：某班级/年级多次测试的成绩变化

## 6. 非功能需求

### 6.1 性能
- 列表页首次加载 < 1s（1000 条数据以内）
- 导出操作异步执行，大数据量时显示进度

### 6.2 安全
- 所有 API 除登录外均需 JWT 认证
- 密码 bcrypt 加密存储
- 敏感操作写入操作日志
- SQL 注入防护（使用参数化查询）

### 6.3 兼容性
- 支持 Chrome/Edge/Firefox 最新两个版本
- 分辨率最低支持 1366x768

### 6.4 数据一致性
- 删除学校前校验是否有关联数据
- 变更学生班级时同步更新 `class_name`
- 删除学生采用软删除，保留历史成绩
